Complete Socket Learning Reference

Use this reference while running the animation. The first table explains address family, the second table explains communication style, and the third table explains protocol selection.

1) Socket Family Table
Family Meaning Default? Address format / example Purpose Common use case
AF_INET IPv4 Internet addresses Yes, most common in teaching ("127.0.0.1", 5000) Network communication using IPv4 Client-server apps, web apps, chat apps, file transfer
AF_INET6 IPv6 Internet addresses No ("::1", 5000) Network communication using IPv6 Modern networks, future-proof systems, large-scale internet services
AF_UNIX Unix local socket No "/tmp/mysocket" Communication between processes on the same machine Local IPC on Linux/macOS, fast communication between services
AF_CAN Controller Area Network No CAN interface and identifier Communication in CAN bus systems Cars, industrial controllers, embedded systems
AF_RDS Reliable Datagram Sockets No Specialized addressing Reliable datagram-based communication Specialized cluster/network systems
AF_PACKET Direct packet-level interface on Linux No Network interface level Access raw packets at link layer Packet sniffing, network analysis, security tools
AF_BLUETOOTH Bluetooth communication No Bluetooth device addresses Wireless communication over Bluetooth IoT devices, wireless peripherals
AF_NETLINK Kernel-user communication on Linux No Kernel interface Communication between user programs and Linux kernel subsystems System/network management tools
2) Socket Type Table
Socket type Meaning Default in examples? Usually maps to Communication style Purpose Common use case
SOCK_STREAM Stream socket Yes TCP Connection-oriented, reliable, ordered Continuous data stream between two endpoints Web servers, chat systems, client-server apps
SOCK_DGRAM Datagram socket No UDP Connectionless, faster, not guaranteed Send separate packets/messages DNS, video streaming, online games
SOCK_RAW Raw socket No Raw IP / packet access Very low-level access Manual packet handling Security tools, packet sniffers, protocol testing
SOCK_SEQPACKET Sequential packet socket No Depends on family Connection-oriented, preserves message boundaries Reliable message-based communication Advanced IPC / special OS-level systems
SOCK_RDM Reliably delivered messages No Rarely used Message-based reliable delivery Specialized reliable messaging Rare / advanced networking
SOCK_NONBLOCK Non-blocking flag No Flag, not main type by itself Operations do not wait Faster async-style behavior High-performance servers
SOCK_CLOEXEC Close-on-exec flag No Flag Auto close on process exec Resource safety Secure system programming
3) Protocol Table
Protocol value / constant Meaning Usually used with Purpose Use case
0 Default protocol chosen automatically Any family/type combo Simplest and most common choice Almost all beginner programs
IPPROTO_TCP TCP SOCK_STREAM Reliable transport Web, chat, client-server apps
IPPROTO_UDP UDP SOCK_DGRAM Fast packet-based transport Streaming, games, DNS
IPPROTO_ICMP ICMP Often raw sockets Control and error messages Ping tools, diagnostics
IPPROTO_RAW Raw IP SOCK_RAW Manual packet creation Security tools, protocol experiments
IPPROTO_IPV6 IPv6 protocol layer IPv6 sockets IPv6-specific handling Advanced IPv6 networking

Easy Explanation For Students

  • AF_INET: use IPv4 address + port.
  • AF_INET6: use IPv6 address + port.
  • AF_UNIX: use a file path instead of IP address.
  • AF_CAN: used in vehicles and embedded systems.
  • Others are more specialized.
  • Most important beginner family: socket.AF_INET.
  • SOCK_STREAM: like a phone call, reliable continuous stream.
  • SOCK_DGRAM: like sending separate letters/messages.
  • SOCK_RAW: direct low-level network internals.

Most Important Beginner Combinations

Combination Meaning
AF_INET + SOCK_STREAM IPv4 TCP socket
AF_INET + SOCK_DGRAM IPv4 UDP socket
AF_INET6 + SOCK_STREAM IPv6 TCP socket
AF_UNIX + SOCK_STREAM Local machine stream socket

Reminder: protocol is usually 0, which means the OS automatically picks the right one.
